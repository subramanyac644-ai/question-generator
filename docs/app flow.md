# App Flow — Question Generator

## 1. Teacher — Generate and Export

```
Upload/select source PDF
        │
        ▼
Select question type(s) + per-type count
 (count = 0 ⇒ type excluded from this run)
        │
        ▼
Trigger generation
        │
        ▼
For EACH selected type, independently:
  generate → validate count → trim if over /
  retry if under → success OR per-type failure
        │
        ▼
Combine all successful types into one result set
 (assign globally unique ids across the full set)
        │
        ▼
Teacher reviews combined set inline
 (sees succeeded types + any per-type failure messages)
        │
        ├──▶ Edit individual question
        ├──▶ Request regeneration of one type
        │
        ▼
"Export Questions" button appears
 (visible once ≥1 type succeeded)
        │
        ▼
Teacher clicks Export
        │
        ▼
System runs export validation (VR-1..VR-4)
        │
   ┌────┴─────┐
   ▼          ▼
 Valid      Invalid
   │          │
   ▼          ▼
Download   Block download +
questions_   show "Invalid question
<timestamp>   structure detected"
.json
```

## 2. HOD — Review and Approve

```
Open department dashboard
        │
        ▼
List of question sets generated by Teachers in department
        │
        ▼
Open a set → review by type
 (per-type counts, any failed types visible)
        │
   ┌────┴─────┐
   ▼          ▼
Approve    Request regeneration
for          of a specific type
publish      (routes back to Teacher)
   │
   ▼
Set becomes visible to Students
(unless self-publish setting bypasses this step)
```

## 3. Principal — Oversight

```
Open institution-wide analytics view
        │
        ▼
See aggregate metrics across departments:
 generation volume, approval rates, export activity
        │
        ▼
Drill into a specific department
        │
        ▼
View department-level aggregates only
 (cannot open raw question content or export files)
```

## 4. Student — Assessment Access

```
Open assigned assessments list
        │
        ▼
Only assessments approved by HOD
 (or self-published Teacher set, if institution allows) appear
        │
        ▼
Open assessment → rendered question view only
 (no raw JSON, no draft sets, no answer key)
        │
        ▼
Submit answers
        │
        ▼
Result / answer key released per institution policy
 (only after submission)
```

## 5. Cross-Role State Diagram (question set lifecycle)

```
[Draft: generating]
        │ all/any type succeeds
        ▼
[Draft: reviewable]──(Teacher edits / regenerates type)──┐
        │                                                 │
        │ Teacher exports (optional, parallel path)       │
        ▼                                                 │
[Exported JSON file]                                       │
        │                                                  │
        ▼ (separate from export) Teacher submits for review│
[Pending HOD approval] ◀─────────────────────────────────┘
        │
   ┌────┴─────┐
   ▼          ▼
Approved   Regeneration requested
   │          │
   ▼          └──▶ back to [Draft: reviewable]
[Published — visible to Students]
```

## 6. Error / Edge-Case Flow (per-type generation)

```
Type requested with count > 0
        │
        ▼
Generate against source content
        │
   ┌────┴────────────────┐
   ▼                      ▼
Insufficient content   Sufficient content
   │                      │
   ▼                      ▼
Per-type failure       Count check
message; other            │
types unaffected    ┌─────┴─────┐
                     ▼           ▼
                  Returned    Returned
                  == requested  != requested
                     │             │
                     ▼          ┌──┴──┐
                  Accept        ▼     ▼
                              > req  < req
                                │      │
                                ▼      ▼
                              Trim   Retry → still short →
                              to     fail with per-type error
                              exact
                              count
```
